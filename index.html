Use case website for web scraping implementation

An undergraduate project work on cybercrime Investigation in Nigeria using data extraction.

To collect information from a website or web-based application, a web scraping system adheres to a few basic principles. To get the web page's HTML or XML, an HTTP request is made to the website. This is accomplished by utilizing a library, such as the Python Requests library. The HTML or XML of the web page is parsed using the Beautiful Soup parser library. By doing this, a parse tree is produced that can be used to search for and retrieve particular elements or characteristics. Using the parser library's methods and properties, the necessary data is extracted from the parse tree. Finding particular tags or characteristics or employing search criteria like class names or ids are required for this. The extracted data is cleaned and preprocessed. Formats are standardized, unnecessary or irrelevant information is deleted, and errors or missing values are handled. The cleaned data is saved in a structured format, like a CSV file.

Model


Phase One

Determine the website or online application involved in the cybercrime


Phase Two

Scrape information from its pages using the Python Requests library


Phase Three

Use Beautiful Soup to parse and extract information from the pages


Phase Four

Save the extracted information in csv format


Phase Five

Document and share csv file for further investigation and analysis
